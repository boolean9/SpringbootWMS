# 项目实现思路与接口梳理

## 一、项目整体实现思路

本项目为典型的前后端分离的仓库管理系统（WMS），后端采用 Spring Boot + MyBatis-Plus 框架，前端为 Vue（未详细分析前端部分）。后端主要实现了用户、菜单、货品、货品类型、仓库、出入库记录等模块的增删改查及业务逻辑。

- **分层结构**：controller（控制器）负责接收前端请求，调用 service（服务层）处理业务逻辑，service 通过 mapper（数据访问层）操作数据库，entity（实体类）映射数据库表结构。
- **通用返回**：所有接口统一返回 `Result` 对象，包含状态码、消息、数据等。
- **分页与查询**：大量接口支持分页和条件查询，参数通过 `QueryPageParam` 封装。
- **自动代码生成**：通过 `CodeGenerator` 类可自动生成基础的 CRUD 代码，提升开发效率。
- **权限与菜单**：用户登录后根据角色分配菜单权限。

## 三、典型业务流程说明

- **用户登录**：`/user/login`，校验账号密码，返回用户信息及菜单权限。
- **货品出入库**：`/record/save`，根据 action 字段判断是入库还是出库，自动调整货品库存数量。
- **分页与条件查询**：所有 listPage 接口均支持分页和多条件组合查询，便于前端灵活展示数据。

---

## 四、技术要点

- **MyBatis-Plus**：简化了 CRUD 操作，支持 Lambda 条件构造器，分页插件等。
- **统一返回结构**：所有接口返回 `Result`，便于前端统一处理。
- **自动填充**：如创建时间等字段自动填充。
- **跨域支持**：通过 `CorsConfig` 实现全局跨域配置。
